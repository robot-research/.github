# RoboEcosystem：机器人学习仿真生态系统

## 1. 项目概览

本项目包含三个核心仓库，构成了一个完整的机器人学习仿真生态系统：

- **RoboSuite**：核心仿真框架，基于MuJoCo提供高性能机器人仿真环境
- **RoboCasa**：基于RoboSuite扩展的厨房场景仿真环境，提供丰富的任务和场景
- **RoboMimic**：模仿学习框架，用于处理和训练从RoboSuite/RoboCasa收集的数据

## 2. 仓库关系与数据流

### 2.1 功能定位

| 仓库 | 主要功能 | 角色 |
|------|---------|------|
| RoboSuite | 机器人仿真核心框架，提供环境、机器人模型、控制器 | 基础仿真平台 |
| RoboCasa | 扩展的厨房场景仿真，提供丰富的任务和数据集 | 场景扩展 |
| RoboMimic | 模仿学习算法实现，数据处理和模型训练 | 学习框架 |

### 2.2 数据流

1. **数据采集**：通过RoboSuite或RoboCasa创建仿真环境，收集人类演示数据
2. **数据处理**：使用RoboMimic的转换工具处理原始数据
3. **模型训练**：在RoboMimic中训练模仿学习模型
4. **策略部署**：将训练好的模型部署到仿真环境中测试

## 3. 核心功能

### 3.1 RoboSuite

RoboSuite采用模块化设计，各模块有对应的基类，具备良好的扩展能力和插拔能力：

- **environments/**：环境定义，包括基础环境类和各种具体任务环境
- **controllers/**：控制器实现，包括复合控制器和各种类型的控制器
- **robots/**：机器人模型定义，包括单臂、双臂、双足、四足等机器人
- **models/**：模型定义，包括场景、物体、机器人等
- **renderers/**：渲染器实现
- **devices/**：遥控设备，支持空间鼠标和键盘控制
- **utils/**：工具函数

### 3.2 RoboCasa

RoboCasa扩展了丰富的厨房场景，提供更真实的家庭环境仿真：

- **datasets/**：预定义的数据集，包括各种厨房任务
- **robocasa/**：核心代码，扩展自RoboSuite
- **examples/**：示例代码和使用案例

### 3.3 RoboMimic

RoboMimic提供了强大的模仿学习能力：

- **algo/**：模仿学习算法实现，如BC、BCQ、CQL、IQL等
- **config/**：配置系统，为不同算法提供配置模板
- **envs/**：环境包装器，适配不同仿真环境
- **models/**：网络模型定义，包括观测网络、策略网络等
- **scripts/**：工具脚本，用于数据处理、训练等

## 4. Sim2Real 技术

工程考虑到仿真与现实的差距，增加了随机化技术，以缩小差距：

### 4.1 视觉随机化
- 相机参数随机化，包括视场角和姿势
- 物体的视觉外观，包括纹理和材质属性
- 光源的光照参数，包括光源属性和姿态

### 4.2 动力学随机化
- 对于物体和机器人的本体参数进行随机化处理

### 4.3 传感器模拟
- 增加了延迟、损坏函数、滤波等用于模拟传感器损坏或延迟等场景

## 5. 安装

参考 Robocasa 工程中的 README.md 进行安装和使用即可。


## 6. 核心特性

- **模块化设计**：三个仓库都采用了清晰的模块化设计，便于扩展和维护
- **功能互补**：从仿真到数据采集再到模型训练，形成了完整的机器人学习 pipeline
- **丰富的功能**：提供了从基础仿真到复杂任务的全面支持
- **活跃的生态**：持续更新和扩展，支持最新的研究需求
- **多机器人支持**：支持单臂、双臂、移动机器人等多种平台
- **真实感渲染**：提供高质量的视觉渲染，增强仿真的真实感

## 7. 结论

RoboSuite、RoboCasa和RoboMimic构成了一个完整的机器人学习生态系统，从仿真环境到数据采集再到模型训练，为机器人学习研究提供了强大的工具支持。它们的模块化设计和扩展性使得研究者可以方便地进行定制和扩展，适应不同的研究需求。

该生态系统不仅支持VLA（Vision-Language-Action）等现代方法，也可以扩展到传统的控制验证策略，如获取仿真环境中的图像数据、机器人本体数据、仿真环境中物体数据等，通过传统的方式进行控制验证。
